# Novata Backend (note-server)

REST API backend for **Novata** ŌĆö a modern, hierarchical note-taking app. Built with Laravel 12, secured with Laravel Passport (OAuth2), and designed to be consumed by a separate frontend application.

## About

Novata lets users organize their notes as nested pages (similar to Notion), mark pages as favorites, and search across their content. This repository (`note-server`) is the API-only backend that powers the app ŌĆö it has no Blade/frontend views of its own and is meant to be consumed by a separate client at the URL configured in `APP_FRONTEND_URL`.

## Tech Stack

- **PHP 8.2+** / **Laravel 12**
- **Laravel Passport** ŌĆö OAuth2 authentication
- **MySQL / SQLite** ŌĆö database
- **Laravel Scout + Meilisearch** ŌĆö full-text search
- **Filament** ŌĆö admin panel
- **Vite + Tailwind CSS** ŌĆö asset bundling
- **Laravel Sail** ŌĆö local Docker development environment

## Features

- User registration with email verification
- Password reset flow
- OAuth2 token-based authentication (Passport)
- Hierarchical pages (create, update, delete, move, icons, nested children)
- Favorites
- Full-text search via Meilisearch/Scout
- Filament admin panel

## Requirements

- PHP 8.2 or higher
- Composer
- Node.js & NPM (for asset building only)
- MySQL or SQLite
- A running Meilisearch instance (for search)

## Installation

```bash
git clone <repo-url>
cd note-server
composer install
cp .env.example .env
php artisan key:generate
```

Configure your database and other settings in `.env`, then run migrations:

```bash
php artisan migrate
```

Install Passport and generate OAuth client credentials (copy the generated ID/secret into `.env` as `APP_PASSPORT_CLIENT_ID` / `APP_PASSPORT_CLIENT_SECRET`):

```bash
php artisan passport:install
```

Install and build frontend assets (Vite/Tailwind, used only for asset compilation):

```bash
npm install
npm run build
```

Start the application:

```bash
php artisan serve
```

Alternatively, use Laravel Sail for a Docker-based environment.

If you use search, make sure a Meilisearch instance is running and reachable at the URL configured in `MEILISEARCH_HOST`.

## Environment Variables

All required variables are listed with safe placeholder values in `.env.example`. A few project-specific ones worth noting:

| Variable                       | Purpose                                                       |
|---------------------------------|----------------------------------------------------------------|
| `APP_FRONTEND_URL`              | URL of the frontend app (used e.g. for email verification redirects) |
| `APP_PASSPORT_CLIENT_ID`        | OAuth client ID generated by `passport:install`                |
| `APP_PASSPORT_CLIENT_SECRET`    | OAuth client secret generated by `passport:install`             |
| `SCOUT_DRIVER`                  | Search driver used by Laravel Scout (`meilisearch`)             |
| `MEILISEARCH_HOST`              | URL of the Meilisearch instance                                 |

## API Documentation

The full API reference (authentication flow, all endpoints, request/response examples) is available in [`docs/API.md`](public/docs/API.md).

Authentication uses Passport's OAuth2 `password` grant type ŌĆö obtain a bearer token from `/oauth/token` and send it as `Authorization: Bearer <token>` on all protected requests.

## Admin Panel

The backend ships with a [Filament](https://filamentphp.com) admin panel, registered via `App\Providers\Filament\AdminPanelProvider`.

- **URL**: `/admin` (e.g. `http://localhost/admin`)
- **Authentication**: session-based login (own login screen), separate from the API's Passport auth. It uses the `web` guard with the `admins` provider, backed by the `Admin` Eloquent model (table `admins`) ŌĆö **not** the regular `User` model used by the public API.
- **Resources**: currently exposes a `UserResource` for managing application users (`App\Models\User`), with list/create/edit pages showing name, email, creation date, and issued Passport tokens.

### Creating the first admin

There is currently no registration route or seeder for admins. Create one manually, e.g. via Tinker:

```bash
php artisan tinker
```

```php
\App\Models\Admin::create([
    'name' => 'Admin',
    'email' => 'admin@example.com',
    'password' => bcrypt('your-strong-password'),
]);
```

Then log in at `/admin` with that email/password.
